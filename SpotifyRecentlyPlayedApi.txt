Spotify Web API: GET /v1/me/player/recently-played
Источник: https://developer.spotify.com/documentation/web-api/reference/get-recently-played
Проверено: 2026-05-18

Назначение
Endpoint возвращает треки из истории недавно прослушанного текущего пользователя.
Spotify отдельно указывает, что podcast episodes сейчас не поддерживаются.

Авторизация
Нужен OAuth scope:
user-read-recently-played

Request
GET https://api.spotify.com/v1/me/player/recently-played

Query parameters
limit:
  Максимальное число элементов.
  Default: 20.
  Minimum: 1.
  Maximum: 50.

after:
  Unix timestamp в миллисекундах.
  Возвращает элементы после этого cursor timestamp, не включая сам cursor.
  Если указан after, before указывать нельзя.

before:
  Unix timestamp в миллисекундах.
  Возвращает элементы до этого cursor timestamp, не включая сам cursor.
  Если указан before, after указывать нельзя.

Ключевые поля для нашего приложения
items[].track:
  Сам трек, который пользователь слушал.

items[].played_at:
  Дата и время, когда трек был проигран.
  Именно это поле нужно использовать, если строить порядок истории по времени.

items[].context:
  Контекст, из которого был проигран трек: playlist, album, artist и т.п.

cursors.after / cursors.before:
  Курсоры для запроса следующей или предыдущей страницы истории.

next:
  URL следующей страницы или null.

Важный вывод
Этот endpoint не возвращает текущую очередь и не гарантирует модель "уже сыгранные треки текущей очереди".
Это именно история недавно прослушанных треков пользователя. Для нашей панели Overlay, если нужен список "что уже прошло в этой сессии/очереди", локальный кеш приложения подходит лучше и делает меньше запросов к Spotify.
