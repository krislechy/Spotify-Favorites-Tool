# Spotify Favorites Tool

Небольшое Windows-приложение для Spotify: по горячей клавише оно получает текущий трек, проверяет, есть ли он в Избранном, добавляет или убирает его и показывает компактное уведомление поверх окон.

Приложение отправляет запросы по горячим клавишам, при авторизации и во время 8-секундного мониторинга смены трека.

На главном экране есть журнал действий: приложение хранит последние 50 событий, показывает время каждого события и коротко пишет, что произошло.

## Запуск

```powershell
dotnet run --project .\SpotifyFavoritesTool\SpotifyFavoritesTool.csproj
```

## Настройка Spotify

1. Открой [Spotify Developer Dashboard](https://developer.spotify.com/dashboard) и создай приложение.
2. В настройках Spotify app добавь Redirect URI:

```text
http://127.0.0.1:53154/callback/
```

3. Скопируй `Client ID`.
4. Запусти приложение, открой настройки, вставь `Client ID`, нажми `Сохранить`, затем `Войти в Spotify`.

Используемые scopes:

```text
user-read-currently-playing user-library-read user-library-modify
```

## Горячая клавиша

Клавиша Избранного задается прямо в настройках приложения. Нажми поле `Клавиша Избранного`, затем нажми нужную кнопку на клавиатуре, включая медиа-кнопки, если Windows отдает их как обычный virtual key.
В настройках сохраняется и код клавиши, и понятное имя для интерфейса, например `VK_MEDIA_STOP`; если имя неизвестно, приложение показывает старый hex-формат вроде `0xB8`.

Отдельно можно назначить `Клавишу статуса Избранного`. По нажатию она получает через Spotify API только текущий трек. Если трек уже есть в кеше, статус Избранного берется из кеша; если трек новый, приложение один раз проверяет статус Избранного, обновляет кеш и показывает уведомление.

При срабатывании:

- если трека нет в Избранном, приложение добавит его и покажет `Добавлено в Избранное`;
- если трек уже есть в Избранном, приложение уберет его и покажет `Убрано из Избранного`.

## Уведомления о смене трека

Если Spotify подключен, приложение раз в 8 секунд проверяет текущий трек. Если URI трека отличается от последнего замеченного трека, приложение берет статус Избранного из кеша всех уже встреченных треков или проверяет его один раз для нового трека, затем показывает уведомление с обложкой, названием, артистом и сердцем.

Если Spotify возвращает `429`, мониторинг останавливается без повторных запросов.

## 429 Too Many Requests

Spotify может ограничивать разные endpoints отдельно. Поэтому получение текущего трека может проходить успешно, а `me/library/contains` для проверки Избранного в это же время может вернуть `429 Too Many Requests`.

Приложение не делает автоматических повторов после `429`, чтобы не добивать лимит:

- показывает уведомление с полным текстом ошибки;
- показывает `Retry-After`, если Spotify его прислал;
- не делает `Task.Delay` и не отправляет повторный запрос;
- кэширует состояние Избранного для текущего трека после успешной проверки или изменения, чтобы не делать повторную проверку без необходимости.

Документация Spotify: [Web API rate limits](https://developer.spotify.com/documentation/web-api/concepts/rate-limits).

## 403 Forbidden

Для текущего трека нужен scope `user-read-currently-playing`, а для добавления и удаления из Избранного нужны `user-library-read` и `user-library-modify`. Если токен был выдан раньше без этих прав, Spotify может возвращать текущий трек, но запрещать изменение Избранного.

Решение: открой настройки приложения, нажми `Выйти`, затем `Войти в Spotify` и подтверди новые права. Приложение использует актуальные Library endpoints:

- `GET /me/library/contains?uris=spotify:track:{id}`;
- `PUT /me/library?uris=spotify:track:{id}`;
- `DELETE /me/library?uris=spotify:track:{id}`.

## Трей

Приложение ведет себя как обычное desktop-приложение: пока главное окно открыто, оно видно на панели задач. Закрытие окна прячет его в системный трей. Двойной клик по значку возвращает окно, правый клик открывает меню `Открыть`, `Настройки`, `Выход`.

## Настройки

Настройки лежат в `%APPDATA%\SpotifyFavoritesTool\settings.json`. При первом запуске после переименования приложение подхватит старый файл из `%APPDATA%\SpotifyRelayOverlay\settings.json` и сохранит его уже в новую папку. Spotify access token и refresh token не сохраняются открытым текстом: приложение шифрует их через Windows DPAPI для текущего пользователя. Старые незашифрованные токены мигрируются при следующем запуске и перезаписываются в защищенном виде.

## RDP и медиа-клавиши

В настройках есть галка `Перехватывать медиа-клавиши локально при RDP`. Когда она включена, приложение регистрирует `VK_VOLUME_*`, `VK_MEDIA_*` и `VK_LAUNCH_MEDIA_SELECT` как локальные глобальные hotkey и дополнительно держит low-level keyboard hook: физическое нажатие гасится до того, как его заберет RDP-клиент, а действие выполняется на локальном ПК. Если медиа-клавиша назначена как действие приложения, выполняется действие Spotify Favorites Tool; остальные медиа-клавиши отправляются локальной Windows как `WM_APPCOMMAND` через shell.

При смене этой галки приложение предлагает перезапуск. Если согласиться, текущий процесс закрывается и новый экземпляр запускается автоматически.
